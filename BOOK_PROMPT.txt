KEY ALGORITHMIC CONCEPTS TO EXPLAIN IN DEPTH
============================================================

1. COMPRESSED SPARSE ROW (CSR) INDEXING
   - Two arrays: segOff[S+1] (offsets) and segMem[N] (listing indices)
   - Listings in segment s: segMem[segOff[s]] through segMem[segOff[s+1]-1]
   - Two-pass construction: count + cumulative sum + populate
   - O(1) lookup, O(k) iteration where k = segment size (independent of N)
   - Same pattern applied at country level (ctyOff/ctyMem)

2. ZERO-COPY TYPEDARRAY CONSTRUCTION
   - JavaScript: new Uint32Array(ArrayBuffer, byteOffset, length)
   - No data copying — the TypedArray is a VIEW over the downloaded bytes
   - Structure-of-Arrays layout: prices[i], predicted[i], profits[i] are in separate contiguous arrays
   - Cache-friendly: sequential access patterns, CPU prefetcher works perfectly
   - 36 MB buffer → 9 TypedArrays constructed in ~20-50 ms

3. ALIGNMENT AND THE makeAlignedU32 HELPER
   - Variable-length segment/country names leave byte offset unaligned
   - Uint32Array constructor requires byteOffset % 4 === 0
   - Solution: buf.slice(byteOffset, byteOffset + count*4) creates aligned copy
   - Only copies CSR data (few KB), not entire 36 MB buffer

4. QUERY EXECUTION PATTERN
   Forward (price one car):
     segId = segIds[pick]
     k = segOff[segId+1] - segOff[segId]   // O(1) segment size
     for p = segOff[segId] to segOff[segId+1]-1:   // O(k)
       i = segMem[p]
       aggregate(prices[i], profits[i], ...)
     return result
   
   Deal discovery (find best deals):
     top = []
     for each segment s:
       sample proportionally to segment size
       for each sampled listing i:
         compute profit score
         insert into top-k heap
     sort top-k
     return result

5. THE TOUCH RATIO (BENCHMARK SCIENTIST'S DREAM METRIC)
   ρ = records_touched / total_records_in_market
   For forward query with k=500 comps and N=1,000,000: ρ = 0.05%
   For full table scan: ρ = 100%
   Advantage: 2,000× less work
   As N → ∞ (market grows), advantage → ∞ (cost stays constant)

=== CODE BLOCK 3: CSR Index Parsing with Alignment Fix ===
```javascript
function parseBinary() {
  const h = META.header_bytes;
  const arrs = META.arrays;
  
  // ... TypedArray construction (see above) ...
  
  // Parse CSR indexes — after variable-length names
  let off = h + arrs.has_images.offset + N;
  const dv = new DataView(BUF);
  
  // Skip segment names (length-prefixed: uint16 len + char[len])
  const sCount = dv.getUint32(off, true); off += 4;
  for (let i = 0; i < sCount; i++) {
    const len = dv.getUint16(off, true);
    off += 2 + len;  // may leave off unaligned!
  }
  
  // Skip country names (length-prefixed: uint8 len + char[len])
  const cCount = dv.getUint32(off, true); off += 4;
  for (let i = 0; i < cCount; i++) {
    const len = dv.getUint8(off);
    off += 1 + len;
  }
  
  // Helper: create aligned Uint32Array from unaligned offset
  function makeAlignedU32(buf, byteOffset, count) {
    const slice = buf.slice(byteOffset, byteOffset + count * 4);
    return new Uint32Array(slice);
  }
  
  // CSR: Segment → Listings
  const soCount = dv.getUint32(off, true); off += 4;
  segOff = makeAlignedU32(BUF, off, soCount); off += soCount * 4;
  const smCount = dv.getUint32(off, true); off += 4;
  segMem = makeAlignedU32(BUF, off, smCount); off += smCount * 4;
  
  // CSR: Country → Listings
  const coCount = dv.getUint32(off, true); off += 4;
  ctyOff = makeAlignedU32(BUF, off, coCount); off += coCount * 4;
  const cmCount = dv.getUint32(off, true); off += 4;
  ctyMem = makeAlignedU32(BUF, off, cmCount);
}
```

=== CODE BLOCK 4: Market Stats via CSR Iteration ===
```javascript
function marketStats(segId, selfIdx) {
  let n = 0, sum = 0, lo = Infinity, hi = -Infinity, sumProfit = 0;
  // Iterate ONLY comps in this segment via CSR
  for (let p = segOff[segId]; p < segOff[segId + 1]; p++) {
    const i = segMem[p];         // listing index
    if (i === selfIdx) continue; // exclude self
    const px = prices[i];
    n++; sum += px;
    if (px < lo) lo = px;
    if (px > hi) hi = px;
    sumProfit += profits[i];
  }
  return {
    n, avg: n ? Math.round(sum / n) : 0,
    lo, hi,
    avgProfit: n ? Math.round(sumProfit / n) : 0
  };
}
```

============================================================

"Every pricing tool answers 'what is this car worth?' — the forward question.
SQL runs forward. The whole industry is built on forward queries.

But the questions that actually make money are:
1. BACKWARD: 'Which of my 500 cars has the highest unrealized profit?'
2. COUNTERFACTUAL: 'What if I price this car 8% lower — will it sell faster?'
3. ABDUCTIVE: 'Why ISN'T this car selling, and what evidence do I need to find out?'

These questions run BACKWARD through the data. SQL cannot answer them
efficiently because they require scanning the entire market.

Our engine answers them by:
- Storing the entire market as fixed-width TypedArrays (36 MB)
- Indexing listings by segment via CSR (O(1) lookup, O(k) iteration)
- Running ALL logic in the browser (zero server costs after initial load)
- Touching only k=500 comps for a forward query vs scanning N=1,000,000

The result: a pricing engine that runs in a browser tab, costs nothing
per query, and answers the backward questions that SQL structurally cannot."
